Module 1: What is HTML?

The Foundation of the Web

HTML stands for HyperText Markup Language. It is not a programming language; it is a markup language used to structure a webpage.

Think of it as the skeleton of a website. It tells the browser exactly where to place text, display images, and place buttons before you add any design styles or colors.

The Golden Anatomy of an HTML Element

HTML relies on components called elements. Most elements consist of an opening tag, the actual content, and a closing tag.

  • Opening Tag: <p> tells the browser where a paragraph begins.
  • Content: The text or media you want to display.
  • Closing Tag: </p> uses a forward slash to tell the browser the element ends here.

Essential Core Tags

  • Headings: Elements from <h1> down to <h6> create text titles. <h1> is the largest and most important.
  • Paragraphs: Defined by the <p> tag to hold standard blocks of text.
  • Links: Created using the anchor <a> tag. They require an attribute called href to specify the URL destination: <a href="https://example.com">Visit Website</a>
  • Images: Embedded using the <img> tag. This is an "empty" or "void" tag, meaning it has no closing tag. It requires a source src attribute to pull the image file: <img src="photo.jpg" alt="A descriptive placeholder text">

Understanding Attributes

Attributes provide extra information about an HTML element. They are always placed inside the opening tag and usually come in name/value pairs like name="value".

  • href: Used in links to define the destination web address.
  • src: Used in images to point to the file path of the picture.
  • alt: Used in images to describe the image for visually impaired users or if the image fails to load.
  • target="_blank": Added to links to force them to open in a brand new browser tab.

Semantic HTML (Writing Meaningful Code)

Modern HTML5 uses structural tags that tell both the browser and search engines exactly what kind of content lives inside them. Instead of using generic boxes everywhere, you should use:

  • <header>: For the top section of your page, typically containing logos and navigation links.
  • <nav>: Reserved specifically for your primary website navigation menus.
  • <main>: Holds the unique, primary content of that specific webpage.
  • <footer>: For the very bottom section, usually containing copyright info and privacy policies.

The Parent-Child Relationship (Nesting)

HTML elements are nested inside one another. The outer element is the parent, and the inner element is the child. You must always close tags in the reverse order that you opened them.

<!-- Correct Nesting -->
<p>This text contains some <strong>bold text</strong> inside it.</p>

<!-- Incorrect Nesting (Do not do this!) -->
<p>This text contains some <strong>bold text.</p></strong>

Code Comments

You can leave hidden notes to yourself or other developers inside your code using comment tags. Browsers completely ignore these, so they will never show up on your live webpage.

<!-- This is a comment. It is invisible to website visitors! -->

Anatomy of a Full Webpage Document

Every standard HTML file must follow a specific, nested skeletal blueprint:

<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="utf-8">
    <title>My First Website</title>
</head>
<body>
    <h1>Welcome to My Website</h1>
    <p>This is where the user-facing content lives.</p>
</body>
</html>
  • <!DOCTYPE html>: Informs the browser that this is a modern HTML5 document.
  • <html>: The root wrapper that contains every single piece of code on the page.
  • <head>: Holds the hidden background settings, character encoding, and browser tab titles.
  • <body>: Contains everything visible to the actual website visitor.

Practice on Your Own Computer

  1. Open a basic text program like Notepad on Windows or TextEdit on Mac.
  2. Copy the full webpage blueprint and paste it into your text document.
  3. Save the file with the exact name index.html (ensure the file extension is strictly .html, not .txt).
  4. Double-click your new file to open it directly inside any web browser to see your work live.
Module 2: How Can You Learn HTML?

The Best Way to Learn

Learning HTML is straightforward because you can see your results instantly. Here is the step-by-step method to master it:

  • Understand the Tags: Learn how elements are built using opening and closing brackets.
  • Practice Writing: Open a plain text editor on your computer (like Notepad or TextEdit) and type the code yourself instead of copy-pasting.
  • Test in a Browser: Save your files as .html and drag them directly into Google Chrome to see your code turn into a live website.

1. Master Browser Developer Tools (The Inspector)

Every website on the internet leaves its HTML skeleton completely exposed. You can study professional code directly inside your web browser.

  • View Page Source: Right-click any webpage and select "View Page Source" to read the raw, static HTML file sent by the server.
  • Inspect Element: Right-click a specific image, button, or text block and select "Inspect". This opens the **DOM (Document Object Model) Tree**, showing you the live, real-time hierarchy of tags.
  • Live Editing: Double-click any tag or text inside the inspector panel. You can change text or delete tags inside your browser tab instantly to see how the website layout reacts without breaking the actual site.

2. Upgrade from Basic Text Editors to a Code Editor

While Notepad and TextEdit work, professional developers use dedicated **IDEs (Integrated Development Environments)** like Visual Studio Code (VS Code) to accelerate their learning. These tools provide:

  • Syntax Highlighting: Colors tags, attributes, and text differently so typos and missing brackets stand out instantly.
  • Auto-Completion (IntelliSense): Automatically types the closing tag for you the moment you type the opening tag's closing bracket.
  • Emmet Shortcuts: Allows you to type shorthand symbols (like typing p and hitting Tab) to instantly generate a full element template.

3. Learn How to Debug Common Syntax Mistakes

Browsers are highly forgiving. If you make a mistake, they will not crash; instead, they guess what you meant, which often scrambles your layout. Look out for these three bugs:

  • The Bleeding Effect: Forgetting a closing tag slash (leaving out </strong>) forces that formatting to bleed into the rest of your document, styling every element below it.
  • Broken Assets: If an image displays as a broken icon, your file path is incorrect. A single typo in a folder name or file extension prevents the browser from finding the source file.
  • Invisible Content: If you accidentally type standard text using the < and > symbols, the browser misinterprets it as a code tag and completely hides it from the visitor.

4. Validate Code with the W3C Standards Tool

It is difficult for a beginner's eyes to spot tiny syntax errors. The **W3C Markup Validation Service** is a free, official industry tool used to grade code accuracy.

You can copy your code and paste it directly into this validator. It scans your document line by line, flags unclosed tags, catches deprecated elements, and gives you specific line numbers to fix.

Module 3: The Basics of HTML Code

Beyond Plain Text: Inline Text Formatting

Once you have basic text blocks, you need to change how specific words look and read. HTML has built-in tags to modify text meaning and physical appearance:

  • Importance: Wrap text in <strong> to make it bold and signal to search engines that the text is crucial.
  • Emphasis: Wrap text in <em> to italicize words, adding emphasis to a sentence.
  • Highlighters: Wrap text in <mark> to apply a bright yellow background highlight behind specific words.
  • Subscript & Superscript: Use <sub> for lower text (like the 2 in H2O) and <sup> for higher text (like footnotes1).
  • Strikethrough: Wrap text in <del> to display a line drawn directly through deleted or outdated information.
  • Underline / Inserted: Wrap text in <ins> to show newly added or underlined information.

Controlling Blank Space and Line Breaks

If you press the Enter key multiple times or type extra spaces in your text editor, web browsers ignore it and smash your words together. To control page breaks, you must use code:

  • Line Breaks: Insert the <br> tag anywhere you want to force text to jump down to a brand-new line. This is an empty tag with no closing tag.
  • Horizontal Rules: Insert the <hr> tag to draw a thin horizontal line across the screen to visually split up different topics.
  • Preformatted Text: Wrap unique content inside the <pre> tag. This rare tag forces the browser to display all spaces, line breaks, and tabs exactly as you typed them in your text file, using a fixed-width typewriter font.

Handling Quotations and Citations Semantically

When pulling quotes or definitions from external references, HTML provides explicit semantic options to formatting text blocks:

  • Block Quotations: Wrap large, multi-line quotes from external sources inside a <blockquote> tag. Browsers automatically indent these blocks from both sides to separate them from standard text.
  • Short Inline Quotes: Wrap short, single-sentence quotes inside a <q> tag. The browser automatically places matching quotation marks around the text for you based on the language.
  • Creative Work Citations: Wrap titles of books, songs, movies, or research articles inside a <cite> tag to properly mark the reference asset. This text typically renders in italics.

Displaying Code, Computer Input, and Abbreviations

When writing technical content, you need specific tags to differentiate regular human sentences from technical information:

  • Inline Code Snippets: Wrap standalone code terms inside a single <code> tag to give them a distinct, fixed-width monospace font appearance.
  • Keyboard Inputs: Wrap physical keys or keyboard shortcuts inside a <kbd> tag to represent layout commands (like <kbd>Ctrl</kbd> + <kbd>S</kbd>).
  • Abbreviations: Wrap acronyms inside an <abbr> tag and use its title attribute to hold the full definition (e.g., <abbr title="World Wide Web">WWW</abbr>). Visitors can hover their mouse over the text to see the full spelling appear in a small pop-up box.

HTML Character Entities (Displaying Protected Symbols)

Because HTML uses the < and > brackets to read code, you cannot simply type them into normal sentences. The browser will get confused and think you are writing code.

To display special characters safely, you must use code abbreviations called character entities:

  • Less Than symbol (<): Type &lt; in your code.
  • Greater Than symbol (>): Type &gt; in your code.
  • Ampersand symbol (&): Type &amp; in your code.
  • Copyright symbol (©): Type &copy; in your code.
  • Trademark symbol (™): Type &trade; in your code.
  • Registered Trademark (®): Type &reg; in your code.
  • Non-Breaking Space: Type &nbsp; to force a blank space between two words that you never want split apart by a line break.

Block-Level vs. Inline Display Concepts

HTML elements behave in one of two structural ways on a live screen, which changes how they stack next to each other:

  • Block-Level Elements: Elements like headings, paragraphs, and blockquotes always start on a fresh line and stretch out to consume the entire horizontal width of the webpage layout.
  • Inline Elements: Elements like <strong>, <code>, or character entities only take up as much space as the text inside them. They sit tightly side-by-side within the same line without breaking the paragraph flow.

Module 4: Essential Script Elements

Connecting External Scripts and Styles

A true webpage relies on more than just static content. To make your site interactive or styled, you must link it to external files using specific resource tags:

  • <script> (The Brains): Used to embed or point to an external JavaScript file. This allows you to add logic, math calculators, animations, or interactive elements to your page.
  • <link> (The Wardrobe): Placed inside the hidden <head> section to connect an external CSS stylesheet. It tells the browser where to find your colors, fonts, and layouts. It is an empty tag with no closing pair.

Anatomy of Script and Link Integration

To connect these background engines, you use specific attributes inside your tags to guide the browser straight to your code files:

<!-- Linking an external CSS stylesheet -->
<link rel="stylesheet" href="style.css">

<!-- Linking an external JavaScript file -->
<script src="script.js"></script>
  • rel="stylesheet": Tells the browser that the linked document is specifically a style document.
  • src="script.js": Tells the tag exactly where your JavaScript logic file lives on your computer folder.

Script Execution Speed: Defer vs. Async Attributes

When a browser loads a web page, it reads the HTML from top to bottom. If it runs into a massive <script> tag, it will completely pause rendering your layout until that script finishes downloading. To prevent this slow behavior, HTML provides two optimization attributes:

  • defer: Tells the browser to continue downloading the script in the background but wait to execute it until the HTML layout is completely parsed and finished loading. This is the safest default choice.
  • async: Tells the browser to download the script in the background and run it the exact millisecond it finishes downloading, even if it interrupts the HTML loading process. This is perfect for independent third-party code like analytics counters.
<!-- Safe background loading -->
<script src="script.js" defer></script>

<!-- Independent, immediate loading -->
<script src="analytics.js" async></script>

Inline Scripting vs. External Scripting

Just like text formatting, you can write script commands in two distinct locations depending on how large your web project becomes:

  • Inline Scripts: You can type JavaScript code directly between an opening and closing <script> tag right on your current page for tiny, quick tasks.
  • External Scripts: For clean coding, you leave the script tag empty and use the src attribute to point to a completely separate .js file. This keeps your layout code and logic code isolated.

Advanced Script Security: SRI and Crossorigin

When you link to a script hosted on a third-party server (like an external library), you expose your website to security risks. If hackers compromise that external server, they can inject malicious code into your website. HTML5 protects against this using **Subresource Integrity (SRI)**:

  • integrity: Contains a long, encrypted security hash string. The browser checks this hash against the script file it downloads. If a single character of the script has been altered, the browser blocks it instantly.
  • crossorigin: Establishes a secure connection request when fetching scripts from entirely separate domain names.
<!-- Secure third-party script inclusion -->
<script src="https://example.com" 
        integrity="sha384-oqVuAfXRKap7fdgcCY5uykM6+R9GqQ8K/ux" 
        crossorigin="anonymous"></script>

Data Blocks: Using type="application/json"

Sometimes you need your HTML file to hold raw structured data that JavaScript can read later, but you do not want it to execute as active code. You can turn a script tag into a raw data locker by changing its type attribute:

<script type="application/json" id="config-data">
{
    "theme": "dark",
    "fontSize": "16px"
}
</script>

The browser reads this configuration content purely as plaintext data data, ignoring it entirely until your JavaScript targets its id attribute to extract the layout settings.

Advanced Resource Hints: Preload and Prefetch

You can use the <link> tag to give the web browser early hints about what assets it will need to download next, dramatically speeding up website page loads:

  • preload: Forces the browser to download a high-priority asset (like a critical font or font file) instantly because it will be needed on the immediate screen layout.
  • prefetch: Tells the browser to download an asset during idle time because the visitor is highly likely to click onto the next webpage that uses it.
<!-- Preloading an urgent layout asset -->
<link rel="preload" href="critical-style.css" as="style">

The Noscript Safety Net

Some users browse the web with JavaScript disabled for privacy reasons, or their browser might fail to load your script file entirely. HTML protects your user experience with a backup tag:

  • <noscript>: Contains plain text or warning messages that *only* appear if JavaScript is broken or turned off. If scripts are running perfectly, the browser hides this tag entirely.
<noscript>
    <p style="color: red;">Warning: This page requires JavaScript to function properly.</p>
</noscript>
